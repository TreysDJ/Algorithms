## Selection Sort
Сортировка выбором. Принцип работы: Изучить массив, начиная с индекса 0. Поменяйте местами следующие два числа: число с индексом 0 и наименьшее число в массиве, начиная с индекса 0. Изучить массив, начиная с индекса 1. Поменяйте местами следующие два числа: число с индексом 0 и наименьшее число в массиве, начиная с индекса 1 и так далее.
```java
    public void sort(int[] arr) {
        for (int i = 0; i < arr.length; i++) { // анализирует все элементы в массиве
            int smallestIndex = smallestIndexOf(arr, i); // находит наименьший от первого элемента в массиве с помощью цикла (в зависимости от шага) 
            if (arr[i] > smallest(arr, i)) { // сравнивает наименьший с первым элементом (ну который зависит от шага), если первый элемент больше - меняет их местами
                arr[smallestIndex] = arr[i];
                arr[i] = smallest(arr, i);
            }
        }
    }
 ```
 Лучший случай - если все элементы в массиве уже будут отсортированы. Сложность - O(n^2), так как чтобы найти наименьший элемент в массиве нужно изучить весь массив (цикл). И если весь массив уже будет отсортирован - алгоритм всё равно будет пытаться найти наименьший элемент. Отсюда и такая сложность
 Худший случай - точно также O(n^2)
 Средний случай - O(n^2)
