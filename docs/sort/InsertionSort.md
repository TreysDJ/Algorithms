## Insertion Sort
 Принцип: делит массив на две части: отсортированную (которая по дефолту состоит из первого элемента неотсортированного массива) и неотсортированную (остальных). Затем берёт ключевой элемент (первый элемент неотсортированного массива) и сравнивает его с элементами из отсортированного массива. И пока "первый элемент" неотсортированноо меньше элементов отсортированного массива, он меняет их местами.
 ```java
 public void sort(int[] arr) {
        for (int i = 1; i < arr.length; i++) {
            int current = arr[i];   
            int index = i;
            while (index > 0 && arr[index - 1] > current) { // условие index > 0 чтобы не вызвало ошибку IndexOfBoundsException и условие пока index-1 меньше первого элемента они меняются местами
                arr[index] = arr[index-1];
                index--; // чтобы сравнить следующий элемент из отсортированного массива уменьшаем index
            }
            arr[index] = current;
        }
    }
```
Лучший случай - если все элементы в массиве уже будут отсортированы, внутренний цикл не будет задействован так, как он был бы использован при неотсортированном массиве, поэтому сложность - O(n).
Худший случай - задействует внутренний цикл while, сложность - O(n^2)
Средний случай - O(n^2)
