Рекурсия:
## 1. Что такое рекурсия. Что такое базовый случай. Без базового случая рекурсия бесконечна. Принцип работы рекурсии. Где выделяется память для следующего вызова рекурсивной функции (читай про стек)
    - записать все в документ как всегда 
    **Рекурсия** - метод использования функции с целью решения задачи через сведение её к самой себе в более простых условиях. Базовый случай (простой случай) - можно сравнить с условием выхода из цикла. Память для следующего вызова рекурсивной функции выделяется в стеке, если стек будет заполнен - будет ошибка переполнения.
  ```java
     public int factorial (n) {
        if (n == 1) {
            return 1; // простой случай
        }
        return n * factorial(n - 1); // сведение к простому случаю
        }
 ```
## 2. Имплементировать алгоритм нахождения N-ого числа Фибоначчи с помощью рекурсии 
    - дай мне пошаговый пример работы рекурсии и ее разворачивания обратно для любого числа. Например хотим найти 5 число в последовательности фибоначчи. Картинка / диаграмма нужна где видно дерево вызова функций и как всё сворачивается обратно и находится решение
 ```java
     private int fibonachi(int number) {
        if (number < 0) {
            return -1;
        }
        if (number == 0) {
            return 0;
        }
        if (number == 1) {
            return 1;
        }
        return fibonachi(number - 1) + fibonachi(number - 2);
    }
 ```
    - Также напиши альтернативную версию алгоритма с использованием цикла вместе рекурсии
 ```java
     public int fibonachi(int n) {
        if (n < 0) {
            return -1;
        }
        if (n == 0) {
            return 0;
        }
        if (n == 1) {
            return 1;
        }
        int[] fibonacci = new int[n + 1];
        fibonacci[0] = 0;
        fibonacci[1] = 1;

        for (int i = 2; i <= n; i++) {
            fibonacci[i] = fibonacci[i - 1] + fibonacci[i - 2];
        }
        return fibonacci[n];
    }
 ```
    - к каждой имплементации - тест
    - не касается напрямую рекурсии, но почитай что такое "мемоизация" и как это касается рекурсии вообще. Это понадобится тебе на олимпиадах
    Мемоизация - вариант кеширования, заключающийся в том, что для функции создаётся таблица результатов, и при вычислениии какого-либо результата такой результат законсится в таблицу. И если для дальнейших вычислениях будет нужен уже существующий результат, он берётся из таблицы. 
    Её можно применить только к чистым функциям:
     1. детерминированным (при одном и том же наборе параметров функция должна вовзвращать одинаковое значение) 
     2. без побочных эффектов функции (Операции ввода/вывода, изменение глобальных переменных (которая не является переменной этой функции), изменение внутреннего состояния объекта, на который указывает аргумент)
```java
    public HashMap<Integer, Integer> fibonacheri = new HashMap<>();
    public int fibonachi(int number) {
        if (number < 0) {
            return -1;
        }
        if (number == 0) {
            return 0;
        }
        if (number == 1) {
            return 1;
        }
        if (fibonacheri.containsKey(number)) {
            return fibonacheri.get(number);
        }
        int result = fibonachi(number - 1) + fibonachi(number - 2);
        fibonacheri.put(number, result);
        return result;
    }
```
